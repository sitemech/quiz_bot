# Архитектура NestQuiz

## Обзор системы

NestQuiz состоит из нескольких основных модулей, работающих параллельно:

```
┌─────────────────┐
│  Telegram Bot   │ ← Обработка команд пользователей
│   (main2.py)    │
└────────┬────────┘
         │
         ├──→ Парсинг расписания
         │   └──→ BeautifulSoup + requests
         │
         ├──→ Планировщик уведомлений
         │   └──→ schedule (ежедневно в 05:00)
         │
         └──→ Управление подписчиками
             └──→ JSON файл (subscribers.json)
```

## Основные функции

### Парсинг данных

#### `fetch_quiz_schedule() -> List[Tuple]`
Запрашивает и парсит HTML-страницу с расписанием квизов.

**Возвращает:**
```python
[
    (datetime, name, venue_name, address, time, description),
    ...
]
```

**Процесс:**
1. HTTP GET запрос с User-Agent заголовком
2. Парсинг HTML через BeautifulSoup
3. Поиск блоков `schedule-column`
4. Извлечение данных из каждого блока
5. Формирование кортежей с информацией

**Обработка ошибок:**
- Таймаут 15 секунд
- Проверка HTTP статуса
- Логирование предупреждений при отсутствии данных

#### `parse_date(date_str: str) -> datetime`
Парсит строку даты формата "14 июня, пятница" в объект datetime.

**Особенности:**
- Автоматическое определение года (текущий или следующий)
- Обработка перехода через год
- Словарь месяцев на русском языке

#### `format_date_label(date_obj: datetime) -> str`
Форматирует дату для отображения:
- "🔥 Сегодня" - для сегодняшней даты
- "⚡ Завтра" - для завтрашней даты
- "dd.mm.yyyy" - для остальных дат

### Управление данными

#### `get_today_and_tomorrow_quizzes() -> List[Tuple]`
Фильтрует квизы на сегодня и завтра из полного расписания.

**Логика:**
1. Получает текущую дату (или DEBUG_DATE)
2. Запрашивает полное расписание
3. Фильтрует по датам (сегодня, завтра)
4. Возвращает список

#### `get_next_quiz() -> Optional[Tuple]`
Находит ближайший квиз от текущей даты.

**Алгоритм:**
1. Получает все квизы
2. Фильтрует будущие (дата >= сегодня)
3. Сортирует по дате
4. Возвращает первый элемент

### Управление подписчиками

#### `load_subscribers() -> None`
Загружает список подписчиков из JSON файла при старте.

**Формат файла:**
```json
[124974400, 123456789, ...]
```

#### `save_subscribers() -> None`
Сохраняет текущий список подписчиков в JSON файл.

**Вызывается:**
- При подписке нового пользователя
- При изменении списка

### Уведомления

#### `send_daily_notification() -> None`
Основная функция отправки уведомлений.

**Процесс:**
1. Проверка флагов (`NOTIFICATIONS_ENABLED`, наличие подписчиков)
2. Получение квизов на сегодня и завтра
3. Фильтрация (исключение "[новички]")
4. Формирование сообщения
5. Отправка всем подписчикам

**Формат сообщения:**
```
📅 Ближайшие квизы:

🎲 Название квиза (🔥 Сегодня — 29.11.2025)
⏰ Время: 20:00
📍 Место: Название заведения, Адрес

📝 Описание игры:
Текст описания...
```

### Обработчики команд

#### `/start` или `/subscribe`
```python
@bot.message_handler(commands=['start', 'subscribe'])
def subscribe(message):
```

**Действия:**
1. Добавление chat_id в подписчики
2. Сохранение в файл
3. Поиск ближайшего квиза
4. Отправка информации пользователю

#### `/next`
```python
@bot.message_handler(commands=['next'])
def send_next_quiz(message):
```

**Действия:**
1. Поиск ближайшего квиза
2. Отправка информации пользователю
3. Обработка ошибок с информативным сообщением

#### `/pause` и `/resume`
Управление глобальным флагом `NOTIFICATIONS_ENABLED`.

### Планировщик

#### Режим продакшн
```python
schedule.every().day.at("05:00").do(send_daily_notification)
```
- Запуск в отдельном потоке
- Проверка каждую секунду
- UTC время

#### Режим отладки
```python
while True:
    send_daily_notification()
    time.sleep(10)
```
- Для тестирования
- Уведомления каждые 10 секунд

## Потоки выполнения

```
Main Thread
├── Telegram Bot Thread (infinity_polling)
│   ├── Обработка /start
│   ├── Обработка /next
│   ├── Обработка /pause
│   └── Обработка /resume
│
└── Scheduler Thread (daemon)
    └── Ежедневная проверка и отправка уведомлений
```

## Структура данных

### Кортеж квиза
```python
(
    datetime,      # Дата и время квиза
    str,           # Название квиза
    str,           # Название заведения
    str,           # Адрес
    str,           # Время (например, "20:00")
    str            # Описание игры
)
```

### Множество подписчиков
```python
CHAT_IDS = {124974400, 123456789, ...}  # set[int]
```

## Обработка ошибок

### Уровни обработки

1. **HTTP запросы:**
   - Таймаут 15 секунд
   - `response.raise_for_status()`
   - Логирование ошибок

2. **Парсинг HTML:**
   - Проверка наличия элементов
   - Обработка отсутствующих данных
   - Предупреждения в логах

3. **Telegram API:**
   - Try-except при отправке сообщений
   - Логирование ошибок для каждого чата
   - Продолжение работы при ошибках

4. **Файловые операции:**
   - Проверка существования файлов
   - Обработка исключений JSON
   - Инициализация пустых структур

## Конфигурация

### Переменные окружения
- `BOT_TOKEN` - токен Telegram бота

### Константы в коде
- `DEBUG` - режим отладки
- `DEBUG_DATE` - подмена даты для тестирования
- `URL` - URL расписания
- `REQUEST_HEADERS` - HTTP заголовки
- `SUBSCRIBERS_FILE` - путь к файлу подписчиков

## Логирование

**Уровни:**
- `INFO` - основная информация
- `WARNING` - предупреждения
- `ERROR` - ошибки
- `EXCEPTION` - исключения с трейсбеком

**Формат:**
```
%(asctime)s - %(levelname)s - %(message)s
```

**События:**
- Запуск/остановка бота
- Подписка/отписка пользователей
- Отправка уведомлений
- Ошибки парсинга
- Ошибки Telegram API

## Производительность

### Оптимизации

1. **Множество для подписчиков:**
   - O(1) добавление/проверка
   - Быстрый доступ

2. **Кэширование:**
   - Расписание запрашивается при необходимости
   - Можно добавить кэш с TTL

3. **Многопоточность:**
   - Бот и планировщик в отдельных потоках
   - Неблокирующая работа

### Потенциальные улучшения

- Кэширование расписания (TTL 1 час)
- Асинхронные HTTP запросы
- База данных вместо JSON
- Webhook вместо polling
- Очередь сообщений для массовой рассылки

## Безопасность

### Текущие меры

1. **Токен бота:**
   - Хранение в `config.py`
   - Поддержка переменных окружения
   - Не коммитится в репозиторий

2. **User-Agent:**
   - Имитация браузера
   - Обход простых блокировок

### Рекомендации

- Использовать переменные окружения для токена

- Валидация входящих данных
- Мониторинг ошибок
